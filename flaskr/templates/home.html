Shortest Path Operation Microservice

The Shortest Path Operation Microservice is based on the Flask template. A new blueprint is created in the blueprints folder, the shortest_path.py file, which holds the functionality of the Shortest Path Microservice. When running the Microservice, the functionality can be accessed via the following endpoint.

op/shortest-path

The endpoint has a GET and POST method. The GET response of the endpoint returns a JSON object that specifies which methods and data types are supported by the endpoint. In this case, an IFC-file must be sent in a POST request to the endpoint. The endpoint then returns a JSON file containing all the information about the calculated shortest path. The start and end space also need to be give in the post request to make a correct calculation of the shortest path as well as some additional parameters. The endpoint will thus need to receive a form-data request containing the IFC-file and the additional parameters.

  • ifcFile: The IFC-file
  • start: The start space of the path that needs to be calculated.
  • finish: The finish space of the path that needs to be calculated.
  • accuracy: The length of steps in meters.
  • display: A boolean indicating whether or not a graphical display needs to be returned.
