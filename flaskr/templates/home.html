QTO (Quantity Takeoff) Operation Microservice

The QTO Operation Microservice is based on the Flask template. A new blueprint is created in the blueprints folder, the qto.py file, which holds the functionality of the QTO Microservice. When running the Microservice, the functionality can be accessed via the following endpoint.

op/qto

This endpoint has a GET and POST method. The GET response of the endpoint returns a JSON object that specifies which methods are supported by the endpoint, in this case a GET and POST method. The JSON object also specifies the data type each method accepts and returns (given as its MIME type). For this endpoint, an LBD-file, given in the Turtle (Terse RDF Triple Language) syntax which expresses the Resource Description Framework (RDF) format, must be sent in a POST request to the endpoint. The endpoint then returns a JSON-LD file with the calculated QTO. This process can be tested using Postman or a Controller Microservice. The QTO is calculated using SPARQL (SPARQL Protocol and RDF Query Language). SPARQL is a query language for the RDF data format, used to retrieve and manipulate data. The Building Topology Ontology (BOT) and PROPS ontology are used to perform this SPARQL query. Similar SPARQL queries can be made to retrieve other elements present in a building project.
